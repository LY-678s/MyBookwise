# 数据库表结构说明

**数据库**：`bookstoredb`（MySQL 8.0）  
**字符集**：`utf8mb4_unicode_ci`

---

## 目录

| # | 表名 | 中文名 | 说明 |
|---|------|--------|------|
| 1 | `book` | 图书 | 书目与库存 |
| 2 | `bookauthor` | 图书作者 | 每本书最多4位有序作者 |
| 3 | `creditlevel` | 信用等级 | 折扣率与信用额度规则 |
| 4 | `customer` | 顾客 | 账户、余额、信用 |
| 5 | `orders` | 订单 | 订单主表 |
| 6 | `orderdetail` | 订单明细 | 每个订单的图书明细 |
| 7 | `shortagerecord` | 缺货记录 | 缺货登记 |
| 8 | `procurement` | 采购单 | 补货采购单主表 |
| 9 | `procurementdetail` | 采购明细 | 采购单图书明细 |
| 10 | `supplier` | 供应商 | 供应商基本信息 |
| 11 | `supplierbook` | 供应商图书 | 供应商与图书对应关系 |
| 12 | `customer_update_queue` | 客户更新队列 | 异步处理客户余额/消费更新（内部用） |

---

## 1. `book` — 图书

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `ISBN` | varchar(20) | NOT NULL | — | **主键**，国际书号 |
| `Title` | varchar(100) | NOT NULL | — | 书名 |
| `Publisher` | varchar(100) | NULL | — | 出版社 |
| `Price` | decimal(10,2) | NOT NULL | — | 定价，CHECK > 0 |
| `Keywords` | varchar(200) | NULL | — | 关键字（逗号分隔） |
| `CoverImage` | longblob | NULL | — | 封面图片（二进制） |
| `StockQty` | int | NOT NULL | 0 | 库存数量，≥ 0 |
| `Location` | varchar(50) | NULL | — | 库存位置（如 A区-01架） |
| `MinStockLimit` | int | NOT NULL | 10 | 最低库存限制，CHECK > 0；低于此值触发缺货记录 |

**约束**：`Price > 0`，`MinStockLimit > 0`

---

## 2. `bookauthor` — 图书作者

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `ID` | int | NOT NULL | AUTO_INCREMENT | **主键** |
| `ISBN` | varchar(20) | NOT NULL | — | **外键** → `book.ISBN`（级联删除） |
| `AuthorName` | varchar(50) | NOT NULL | — | 作者姓名 |
| `AuthorOrder` | tinyint | NOT NULL | — | 作者序位，CHECK 1~4 |

**约束**：`AuthorOrder BETWEEN 1 AND 4`  
**说明**：一本书可有最多4位有序作者，通过 `AuthorOrder` 排序。

---

## 3. `creditlevel` — 信用等级

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `LevelID` | int | NOT NULL | — | **主键**，等级编号（1~5） |
| `DiscountRate` | decimal(3,2) | NOT NULL | — | 折扣率，CHECK 0 < x ≤ 1（如 0.90 = 九折） |
| `CanUseCredit` | tinyint(1) | NOT NULL | 0 | 是否可使用信用支付：0=否，1=是 |
| `CreditLimit` | decimal(10,2) | NOT NULL | 0.00 | 信用额度上限，CHECK ≥ 0 |

**内置数据**：

| LevelID | 折扣 | 信用支付 | 信用额度 | 升级条件（TotalSpent） |
|---------|------|---------|---------|----------------------|
| 1 | 九折 | 不可 | 0 | < ¥1000 |
| 2 | 八五折 | 不可 | 0 | ≥ ¥1000 |
| 3 | 八五折 | 可 | ¥500 | ≥ ¥2000 |
| 4 | 八折 | 可 | ¥1000 | ≥ ¥5000 |
| 5 | 七五折 | 可 | 无上限 | ≥ ¥10000 |

---

## 4. `customer` — 顾客

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `CustomerID` | int | NOT NULL | AUTO_INCREMENT | **主键** |
| `Username` | varchar(50) | NOT NULL | — | 登录用户名，UNIQUE |
| `Password` | varchar(50) | NOT NULL | — | 密码（明文，课设简化） |
| `Name` | varchar(50) | NOT NULL | — | 真实姓名 |
| `Address` | varchar(200) | NULL | — | 收货地址 |
| `Email` | varchar(100) | NULL | — | 邮箱，UNIQUE，CHECK 格式含 @和. |
| `Balance` | decimal(10,2) | NOT NULL | 0.00 | 账户余额，CHECK ≥ 0 |
| `LevelID` | int | NOT NULL | 1 | **外键** → `creditlevel.LevelID` |
| `CreditLimit` | decimal(10,2) | NOT NULL | 0.00 | 当前信用额度上限（从等级规则复制） |
| `UsedCredit` | decimal(10,2) | NOT NULL | 0.00 | 已使用信用额度，CHECK UsedCredit ≤ CreditLimit |
| `TotalSpent` | decimal(12,2) | NOT NULL | 0.00 | 累计消费总额（余额支付部分），用于信用升级判断，CHECK ≥ 0 |
| `RegisterDate` | datetime | NOT NULL | CURRENT_TIMESTAMP | 注册时间 |

**约束**：`Balance ≥ 0`，`TotalSpent ≥ 0`，`UsedCredit ≥ 0`，`UsedCredit ≤ CreditLimit`，Email 格式校验

---

## 5. `orders` — 订单

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `OrderID` | int | NOT NULL | AUTO_INCREMENT | **主键** |
| `OrderNo` | varchar(30) | NOT NULL | — | 订单号，UNIQUE（格式：YYYYMMDDNN） |
| `OrderDate` | datetime | NOT NULL | CURRENT_TIMESTAMP | 下单时间 |
| `CustomerID` | int | NOT NULL | — | **外键** → `customer.CustomerID`（级联删除） |
| `ShipAddress` | varchar(200) | NOT NULL | — | 发货地址 |
| `TotalAmount` | decimal(10,2) | NULL | — | 应付金额（折扣后），触发器/应用层计算，CHECK ≥ 0 |
| `ActualPaid` | decimal(10,2) | NOT NULL | 0.00 | 实际已付金额，CHECK ≥ 0，CHECK ≤ TotalAmount |
| `PaymentStatus` | tinyint | NOT NULL | 0 | 付款状态（见下表） |
| `Status` | tinyint | NOT NULL | 0 | 订单状态（见下表） |

**订单状态（Status）**：

| 值 | 含义 |
|----|------|
| 0 | 已下单（待发货） |
| 1 | 已发货 |
| 2 | 已完成（顾客确认收货） |
| 4 | 已取消 |

**付款状态（PaymentStatus）**：

| 值 | 含义 |
|----|------|
| 0 | 未付款 |
| 1 | 已全额支付 |
| 2 | 待补款（信用支付，余额不足部分用信用额度垫付） |
| 3 | 已退款 |

---

## 6. `orderdetail` — 订单明细

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `DetailID` | int | NOT NULL | AUTO_INCREMENT | **主键** |
| `OrderID` | int | NOT NULL | — | **外键** → `orders.OrderID`（级联删除） |
| `ISBN` | varchar(20) | NOT NULL | — | **外键** → `book.ISBN` |
| `Quantity` | int | NOT NULL | — | 购买数量，CHECK > 0 |
| `UnitPrice` | decimal(10,2) | NOT NULL | — | 下单时单价（快照），CHECK > 0 |
| `IsShipped` | tinyint(1) | NOT NULL | 0 | 是否已发货：0=未发货，1=已发货 |

**说明**：`UnitPrice` 为下单时书的定价快照，不随书价变动。当某订单所有明细 `IsShipped=1` 时，订单状态应改为已发货。

---

## 7. `shortagerecord` — 缺货记录

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `RecordID` | int | NOT NULL | AUTO_INCREMENT | **主键** |
| `RecordNo` | varchar(30) | NOT NULL | — | 缺货单号，UNIQUE（格式：SR-XXXXXX） |
| `ISBN` | varchar(20) | NOT NULL | — | **外键** → `book.ISBN`（级联删除） |
| `Quantity` | int | NOT NULL | — | 缺货数量，CHECK > 0 |
| `RegDate` | datetime | NOT NULL | CURRENT_TIMESTAMP | 登记时间 |
| `SourceType` | tinyint | NOT NULL | — | 来源类型（见下表），CHECK in (1,2,3) |
| `CustomerID` | int | NULL | — | **外键** → `customer.CustomerID`（SET NULL）；顾客订货触发时记录 |
| `Status` | tinyint | NOT NULL | 0 | 状态（见下表），CHECK in (0,1,2,3) |

**来源类型（SourceType）**：

| 值 | 含义 |
|----|------|
| 1 | 手动登记 |
| 2 | 库存低于 MinStockLimit 自动生成 |
| 3 | 顾客订货超出库存触发 |

**状态（Status）**：

| 值 | 含义 |
|----|------|
| 0 | 待处理 |
| 1 | 已生成采购单 |
| 2 | 已到货（采购完成） |
| 3 | 已取消 |

---

## 8. `procurement` — 采购单

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `ProcID` | int | NOT NULL | AUTO_INCREMENT | **主键** |
| `ProcNo` | varchar(30) | NOT NULL | — | 采购单号，UNIQUE（格式：PC-XXXXXX） |
| `SupplierID` | int | NOT NULL | — | **外键** → `supplier.SupplierID` |
| `RecordID` | int | NULL | — | **外键** → `shortagerecord.RecordID`（SET NULL）；关联主缺货记录 |
| `CreateDate` | datetime | NOT NULL | CURRENT_TIMESTAMP | 创建日期 |
| `UpdateDate` | datetime | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 最后更新日期 |
| `Status` | tinyint | NOT NULL | 0 | 0=未到货，1=部分到货，2=全部到货，CHECK in (0,1,2) |

---

## 9. `procurementdetail` — 采购明细

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `DetailID` | int | NOT NULL | AUTO_INCREMENT | **主键** |
| `ProcID` | int | NOT NULL | — | **外键** → `procurement.ProcID`（级联删除） |
| `ISBN` | varchar(20) | NOT NULL | — | **外键** → `book.ISBN` |
| `ShortageRecordID` | int | NULL | — | **外键** → `shortagerecord.RecordID`（SET NULL）；关联缺货记录 |
| `Quantity` | int | NOT NULL | — | 采购数量，CHECK > 0 |
| `SupplyPrice` | decimal(10,2) | NOT NULL | — | 供货单价，CHECK > 0 |
| `TotalPrice` | decimal(10,2) | NOT NULL | — | 总价（数量×单价），CHECK ≥ 0 |
| `IsReceived` | tinyint(1) | NOT NULL | 0 | 是否已到货：0=未到货，1=已到货 |

**说明**：当 `IsReceived` 变为 1 时，触发器自动补充对应图书的 `StockQty`，并将关联缺货记录状态更新为已到货。

---

## 10. `supplier` — 供应商

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `SupplierID` | int | NOT NULL | AUTO_INCREMENT | **主键** |
| `SupplierCode` | varchar(20) | NOT NULL | — | 供应商编码，UNIQUE |
| `SupplierName` | varchar(100) | NOT NULL | — | 供应商名称 |
| `SupplyLocation` | varchar(100) | NOT NULL | — | 所在地区 |
| `ContactInfo` | varchar(200) | NULL | — | 联系方式 |
| `IsActive` | tinyint(1) | NOT NULL | 1 | 合作状态：1=活跃，0=已终止 |

---

## 11. `supplierbook` — 供应商图书

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `SupplierID` | int | NOT NULL | — | **联合主键** + **外键** → `supplier.SupplierID`（级联删除） |
| `ISBN` | varchar(20) | NOT NULL | — | **联合主键** + **外键** → `book.ISBN`（级联删除） |
| `SupplyPrice` | decimal(10,2) | NULL | — | 供货单价，CHECK > 0 |
| `LastSupplyDate` | datetime | NULL | — | 最近供货日期 |

**主键**：`(SupplierID, ISBN)` 联合主键  
**说明**：记录哪个供应商可以供应哪本书，以及最新供货价格。

---

## 12. `customer_update_queue` — 客户更新队列（内部用）

| 列名 | 类型 | 可空 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | bigint | NOT NULL | AUTO_INCREMENT | **主键** |
| `order_id` | bigint | NOT NULL | — | 关联订单 ID |
| `customer_id` | bigint | NOT NULL | — | 关联顾客 ID |
| `event_type` | varchar(32) | NOT NULL | — | 事件类型：`deduct_diff` / `refund` / `complete_add` |
| `amount` | decimal(12,2) | NULL | — | 金额 |
| `created_at` | datetime | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| `processed_at` | datetime | NULL | — | 处理时间 |
| `processed` | tinyint(1) | NULL | 0 | 是否已处理 |
| `error_text` | varchar(512) | NULL | — | 错误信息（处理失败时记录） |

**说明**：为解决触发器并发问题而设的异步队列，由存储过程 `process_customer_update_queue` 消费。目前业务逻辑已移至 Django 应用层处理，此表保留作历史兼容。

## 外键关系图（简要）

```
creditlevel ←── customer ←── orders ←── orderdetail ──→ book
                                 │                         ↑
                         shortagerecord ←── procurement    │
                                 │           ↓             │
                                 └─── procurementdetail ───┘
                                                            
supplier ←── supplierbook ──→ book
         └── procurement
         
book ←── bookauthor
```

## 视图（Views）

| 视图名 | 说明 |
|--------|------|
| `v_bookcatalog` | 图书目录：ISBN、书名、出版社、价格、库存、第一作者 |
| `v_customerorders` | 顾客订单详情：订单+顾客+明细+书名的联合视图 |
| `v_stockalert` | 库存预警：StockQty < MinStockLimit 的图书，含告警级别和建议采购量 |

## 存储过程（Stored Procedures）

| 过程名 | 说明 |
|--------|------|
| `p_CreateOrder` | 创建订单：检查库存、计算折扣、创建订单主表和明细、处理缺货 |
| `process_customer_update_queue` | 处理客户更新队列（余额扣减/退款/消费累计） |
